# Internal design documents

These are the **bootstrap design artifacts** — what the project was meant to be
before any code was written:

- [`requirements.md`](requirements.md) — product requirements (what & why)
- [`spec.md`](spec.md) — technical specification (how)

They are **not** living documentation. They reflect the design intent at
`v0.1.0` and are only updated on major architectural shifts. For current
behaviour see:

- [`../../CHANGELOG.md`](../../CHANGELOG.md) — what shipped, when
- [`../architecture.md`](../architecture.md) — current architecture (live)
- the code itself — source of truth

If you're a new contributor and want to understand the *reasoning* behind
the project's shape — read these. If you want to know what the code does
right now — read the code or the CHANGELOG.
